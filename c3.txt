Using fs::read() loads the entire file into memory before scanning. For what’s primary use case (large binaries), this can cause unnecessary memory spikes and slowdowns, especially when scanning many files. Consider scanning via a buffered stream (e.g., File + BufReader) with a small sliding window so peak memory stays bounded while preserving the byte-oriented behavior.
