sigint_cleanup_handler calls Mutex::try_lock() from inside a SIGINT handler. Rust’s standard Mutex (pthread mutex) operations are not async-signal-safe, so this can deadlock or exhibit undefined behavior if SIGINT arrives while the mutex is held or while the allocator/runtime is in an inconsistent state. The handler should avoid any locking or non-async-signal-safe Rust code paths; consider using an async-signal-safe data structure (e.g., a preallocated lock-free ring of C string pointers) or switching to a design where the handler only sets an atomic flag / writes to a self-pipe and cleanup happens in normal control flow.
